📋 CRISP-ML(Q) Executive Summary

This project follows the Cross-Industry Standard Process for Machine Learning with Quality Assurance (CRISP-ML(Q)) framework to build a predictive system for patient appointment no-shows.

Problem Statement

Missed medical appointments result in financial losses, resource misallocation, and longer wait times. Approximately 20% of scheduled appointments result in no-shows, costing healthcare systems billions annually.

Our Approach

  • Data Preparation: Cleaned ~110 K records, engineered temporal features (WaitingTime, DayOfWeek, Month), encoded categoricals, and normalised numerics.
  • Model Engineering: Trained and compared four algorithms — Logistic Regression, Decision Tree, Random Forest, and XGBoost.
  • Evaluation: Selected the highest-accuracy model to optimise overall prediction correctness on unseen data.
  • Deployment: Flask-based web application with a real-time prediction interface for hospital staff.

Key Findings

  • WaitingTime is the single most important predictor — longer gaps between booking and appointment increase no-show risk.
  • Age and SMS reminders are the next most impactful features.
  • Neighbourhood-level patterns suggest accessibility plays a significant role.

Overfitting occurs when a model memorises the training data instead of learning generalisable patterns. We compare training accuracy vs. test accuracy — a large gap indicates overfitting.

Interpretation

A gap of < 5 percentage points between train and test accuracy suggests the model generalises well. Decision Trees tend to overfit without depth limits, whereas Logistic Regression is often too simple to overfit.
